# JET (Journal Entry Testing) 工具重構專案

## 專案背景與目標

### 背景
目前事務所設計的 Journal Entry Testing Tool (簡稱JET) 依賴於 Caseware IDEA 上建立的 VBA 程式，結合簡易 Windows Forms 為介面。目前要重新設計出替代方案，讓分錄測試程序可以脫離 Caseware IDEA 的依賴，並運行於審計員更習慣的 Excel 程式。

### 目標
- 開發功能更聚焦、流程更標準的工具
- 取代對 IDEA 的依賴，降低成本，並增加技術彈性
- 優化 JE 流程，減少人工作業時間，提高準確性
- 提升 JE 服務品質，增強事務所競爭力
- 嘗試 Excel VBA + Access 架構的解決方案

## 技術選擇與架構

### 選定技術：Excel VBA + Access
- 利用事務所的微軟 Office 生態，直接運行於本地電腦
- 限制案件資料筆數上限為一百萬筆
- 避免依賴外部資料庫，故優先使用 Access 資料庫
- 一般審計員更熟悉 Excel 介面，降低學習曲線

### 系統架構
- **前端**：Excel VBA (UserForms、Excel Worksheet)
- **後端**：Access 資料庫 (.accdb)
- **資料交互**：前期開發使用 DAO (ACE) 後期需兼容 ADO (SQL)
- **分層設計**：
  - View：
  - Controller：協調流程，處理觸發事件並執行
  - Manager：管理員，負責
  - Service：定義邏輯與功能的服務
  - Data Access：封裝資料庫操作

## 核心功能需求

### 依照執行流程
1. 匯入資料
2. 驗證資料
3. 篩選資料
4. 匯出底稿

### 1. 資料匯入
- 支援多種資料格式（csv, txt, xls）匯入
- 儲存於資料庫中，後續皆已資料庫進行ETL
- 提供欄位映射機制（Field Mapping），解決不同資料來源欄位命名差異問題

### 2. 驗證資料 
- 科目代碼與名稱標準化（Account Mapping）
- 將相同性質科目分組，便於篩選分析
- 試算表（TB）與分錄（GL）自動比對與驗證
- 借貸平衡測試
- 科目變動金額比對（考慮 1-3 與 4-7 開頭科目特性）
- 自動識別與追蹤差異項目

### 3. 資料篩選
- 預設風險篩選條件（1-3個核心情境）
- 包含日期篩選（例如週末/假日）
- 自定義篩選條件功能 (SQL查詢)
- 條件組合功能（AND/OR 邏輯），使用者可依需求自行組合不同篩選條件，包括金額篩選、編制人員篩選等條件

### 4. 匯出底稿
- 標準化 Excel 工作底稿
- 包含各步驟完整性測試結果
- 高風險分錄識別結果
- 符合審計工作要求的格式與內容

## 工作流程設計

### 1. 資料匯入與準備
1. 匯入原始資料（GL、TB）
2. 欄位映射與標準化名稱
3. 資料預處理（格式統一、項次生成等）

### 2. 完整性測試
1. 試算表與分錄比對
2. 借貸平衡測試
3. 差異項目識別與分析
4. 差異調節建議

### 3. 科目配對與分類
1. 科目代碼與標準科目名稱配對
2. 科目分類與分組
3. 科目特性標記（例如資產、負債、收入等）

### 4. 篩選條件設定
1. 自定義 SQL 篩選條件
2. 條件組合與優先順序設定

### 5. 結果分析與報告生成
1. 篩選結果預覽
2. 工作底稿自動生成
3. 差異說明與調節建議
4. 結果匯出與歸檔

## 專案技術注意事項

1. **資料量處理考量**
   - Excel 限制：單工作表最多 1,048,576 列
   - Access 性能：6萬筆以上可能變慢，需優化查詢
   - 大量資料案件 (>100萬筆) 仍需考慮 IDEA 支援

2. **程式設計原則**
   - 模組化設計，確保各功能獨立
   - 錯誤處理完善，避免運行中斷
   - 狀態保存，防止步驟中斷需重新開始
   - 註解完整，便於維護與擴展

3. **使用者體驗優化**
   - 介面簡潔明瞭，減少操作步驟
   - 進度指示與錯誤提示清晰
   - 預覽功能，確保用戶理解操作結果
   - 操作記錄與復原功能
